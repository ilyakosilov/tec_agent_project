TEC Agent System

LLM tool-calling and multi-agent orchestration for ionospheric TEC analysis. The experiment compares Qwen single-agent, deterministic rule-based multi-agent baseline, and Qwen full LLM multi-agent on identical data, tools, tasks, and GoldRunner metrics.

focus

Orchestration, not generic model quality

The project tests whether agents can use the same deterministic tools correctly under shared evaluation.

oracle

GoldRunner is numerical reference

GoldRunner computes correct structured artifacts after the agent run and is never prompt-visible.

control

Rule baseline passes 5/5

The scripted role workflow validates data, tools, metrics, and artifact flow.

failure mode

Full LLM multi-agent fails on protocol

The current full LLM implementation stalls on role protocol and handoff discipline, not TEC mathematics.

Why handles instead of raw arrays?

Tools return compact handles like series_id, threshold_id, stats_id, and comparison_id. The LLM sees handles and summaries, not full time-series arrays, keeping prompts compact and reproducible.

GoldRunner

Numerical oracle, not an agent

Reference role

GoldRunner computes correct structured artifacts after an agent run. It is not visible to prompts and does not make orchestration decisions.

What it validates

Thresholds, intervals, statistics, pairwise deltas, report artifact completeness and grounding.

Agent architectures

Single-agent, scripted roles, and full LLM roles

Qwen single-agent

One LLM uses textual tool calls and state-aware observations. It does not see expected sequence, GoldRunner or missing goals.

Rule-based multi-agent

Scripted role trajectory: orchestrator -> data -> math -> analysis -> report. It passes 5/5 and validates the decomposition.

Qwen full LLM multi-agent

All roles are model-driven. Runtime validates protocol and tools, but role and tool decisions come from LLM outputs.

Full LLM role contracts

Protocol blocks

Strict XML-like handoff and tool protocol

Anti-patterns: role_response is not a tool; agents are not tools; role names cannot appear as tool names; expected sequences, gold values and missing goals are not passed to prompts.
